Build a complete **Smart Classroom Management System** web application with the following full-stack architecture and AI-powered features. This is a production-ready college classroom management platform for staff, HODs, and students.

---

### 🏗️ **Tech Stack Requirements**

**Frontend:**
- Next.js 14+ with TypeScript
- Tailwind CSS + Shadcn/UI for premium components
- React Query for data fetching
- WebSocket (Socket.io client) for real-time updates
- QRCode.react for QR generation
- React Hook Form + Zod for validation
- Chart.js/Recharts for analytics dashboard

**Backend:**
- Node.js + Express.js (or FastAPI in Python for AI features)
- PostgreSQL with Prisma ORM
- Redis for caching and real-time session management
- Socket.io for WebSocket server
- JWT authentication with refresh tokens
- Firebase Cloud Messaging (FCM) for push notifications

**AI/ML Integration:**
- Python FastAPI microservice for AI features
- Hugging Face Transformers (BERT) for NLP sentiment analysis
- scikit-learn + Google OR-Tools for timetable optimization
- LSTM/Prophet for predictive absence modeling
- Random Forest for burnout risk prediction

**Infrastructure:**
- Docker + docker-compose for containerization
- Deploy on Vercel (frontend) + Railway/Render (backend)
- Environment variables for secrets management

---

### 👥 **User Roles & Authentication**

1. **Staff Login**
   - Email/password authentication with JWT
   - Profile: name, department, designation, phone, department email
   - After login: create/manage "handing subjects" list (subject name, code, type-theory/lab, credits, class sections)

2. **HOD Login**
   - Separate HOD credentials (management role)
   - Dashboard: view all staff, classes, timetable, analytics, burnout alerts
   - Can create classes, allocate staff, set lab schedules, manage breaks

3. **Student Login**
   - QR code-based one-time login (scan QR from staff dashboard)
   - Auto-join class section when scanning QR (e.g., scan → join "2A" class)
   - View weekly timetable, receive broadcast notifications, submit feedback

### 📋 **Core Features to Implement**

#### **1. Staff Subject Management**
- Staff can add/edit/delete subjects they teach
- Each subject has: name, code, type (theory/lab), credit hours, assigned sections
- Store in database with foreign key to staff_id

#### **2. Class Creation & Staff Allocation (HOD)**
- HOD creates class: class_name (e.g., "2A"), department, semester, student_count
- HOD allocates staff to subjects for that class from dropdown of logged-in staff
- Separate allocation for **theory** and **lab** subjects
- Store allocation: class_id, staff_id, subject_id, type (theory/lab)

#### **3. Lab & Theory Schedule Configuration**
- HOD sets **lab time window**: e.g., 8:30 AM – 4:30 PM
- HOD sets **break times**: 
  - Tea break: 10:30–10:45 AM
  - Lunch break: 1:00–2:00 PM
- HOD sets **continuous lab hour limit** (e.g., max 2 hours continuous lab)
- These constraints feed into timetable optimizer

#### **4. Weekly Timetable Generation (AI Optimizer)**
- **Algorithm**: Use Google OR-Tools constraint satisfaction solver
- **Constraints**:
  - No staff assigned to two classes at same time
  - No continuous class >2 hours for same staff → must have 1-hour cooldown
  - Lab sessions only within configured lab window (8:30–4:30)
  - Respect tea/lunch break times (no classes scheduled)
  - Each class gets theory + lab slots per week
  - Balance workload across staff (avoid >20 hrs/week)
- **Output**: JSON weekly timetable (Monday–Friday, time slots)
- **Auto-regenerate**: If staff absence_detected, regenerate and swap slots

#### **5. Staff Real-Time Dashboard**
- Staff logs in → sees their personal weekly timetable
- Click on class slot → navigate to "class details" page
- Display: class name, subject, time, room number, students list
- **Generate QR Code** for attendance:
  - Unique QR per class slot (expires in 2 minutes)
  - One-time use only
  - Geofenced (optional: validate GPS coordinates)
- **Cooldown indicator**: Show if staff is in 1-hour cooldown after 2-hour class

#### **6. Student QR-Based Attendance & Class Joining**
- Student opens app → clicks "Scan QR to Join Class"
- Camera opens → scan QR from staff dashboard
- Auto-join class section (e.g., 2A) → stored in database: student_id, class_id, timestamp
- Student sees: "Successfully joined 2A class" + confirmation
- Student dashboard shows:
  - Entire weekly timetable (same as staff view but student-specific)
  - Attendance history
  - Notifications from HOD/staff

#### **7. Notes Feature (Staff → Classroom)**
- Staff can add notes to any class slot: "Quiz tomorrow", "Bring lab manuals"
- Notes visible to students enrolled in that class
- Timestamped, editable, deletable by staff

#### **8. NLP-Based Student Feedback System**
- HOD opens "feedback window" for specific class + staff (time-bound: 48 hours)
- Students submit anonymous feedback: open-ended text + 5-star rating
- **AI Processing** (Python microservice):
  - Use fine-tuned BERT model for sentiment analysis (positive/negative/neutral)
  - Extract topics: "teaching style", "pacing", "clarity", "engagement"
  - Generate summary: "80% positive, 15% neutral, 5% negative"
  - Store sentiment_score (0–1) per feedback
- Dashboard for HOD:
  - Sentiment trend graph over time
  - Top complaints/praises per staff
  - Red flag alerts if sentiment <0.3 for 2+ weeks

#### **9. Auto Timetable Swap + Broadcast Notifications**
- **Nightly job** (cron at 10 PM):
  - Check for staff absence alerts (see below)
  - If staff unavailable → auto-assign substitute (see #10)
  - Regenerate timetable for next day with new allocation
  - **Version history**: keep old timetable, show changelog
- **Broadcast notifications** (FCM + WebSocket):
  - Notify affected staff: "Your class 2A (Mon 10 AM) swapped with Staff B"
  - Notify students: "Class 2A lab moved to Room 205, Mon 2 PM"
  - Push notification + in-app notification bell

#### **10. Staff Unavailability + Auto Substitute Allocation**
- Staff clicks "Mark Unavailable" (with reason: sick, leave, emergency)
- **Immediate alert** to HOD + department head via email + push notification
- **Auto-substitute algorithm**:
  - Find staff who:
    - Teach same subject (or similar expertise)
    - Are free at that time slot
    - Not in cooldown period
    - Weekly workload <20 hours
    - Lowest recent substitute count (fair distribution)
  - Auto-assign substitute + notify both staff
  - If no substitute found → flag to HOD manually

#### **11. Cooldown Period Enforcement**
- After staff teaches 2+ continuous hours → enter **1-hour cooldown**
- During cooldown: system blocks new class allocation
- Visual indicator on dashboard: "🔥 Cooldown: 45 min remaining"
- Log cooldown violations for burnout analysis

#### **12. Staff Burnout Detection System**
- **Metrics calculated daily** (Python microservice):
  - Weekly teaching hours (sum of all class durations)
  - Consecutive teaching days (≥4 classes/day)
  - Late-day classes (after 4 PM count)
  - Substitute frequency (times covering others/month)
  - Student feedback sentiment score (avg over 2 weeks)
  - Cooldown violations count
- **Burnout risk score** (0–100):
  - Random Forest classifier trained on above features
  - Thresholds:
    - 0–30: Low risk (green)
    - 31–60: Medium risk (yellow) → show warning
    - 61–100: High risk (red) → alert HOD
- **HOD Dashboard**:
  - Heatmap table: staff names vs burnout risk
  - Trend graphs: workload over time per staff
  - Auto-recommendations: "Redistribute 3 hrs from Staff A to Staff B"
  - Export report: PDF of burnout analysis

#### **13. Analytics Dashboard for HOD**
- **Interactive charts**:
  - Workload distribution (bar chart: hours per staff)
  - Room utilization rate (pie chart: occupied vs free hours)
  - Burnout risk heatmap (matrix)
  - Student satisfaction trends (line graph over semesters)
  - Absenteeism patterns (calendar heatmap)
- **Filters**: by department, semester, date range
- **Export**: PDF/CSV reports for meetings

### 🗄️ **Database Schema (Prisma ORM)**

```prisma
model User {
  id        String   @id @default(uuid())
  email     String   @unique
  password  String   // hashed
  role      Role     // STAFF | HOD | STUDENT
  name      String
  department String
  phone     String?
  createdAt DateTime @default(now())
}

model Staff {
  id             String   @id @default(uuid())
  userId         String   @unique
  user           User     @relation(fields: [userId], references: [id])
  designation    String
  subjects       Subject[]
  classesTeaching ClassAllocation[]
  availability   Availability[]
  burnoutScore   Float?
}

model Student {
  id        String   @id @default(uuid())
  userId    String   @unique
  user      User     @relation(fields: [userId], references: [id])
  classId   String
  class     Class    @relation(fields: [classId], references: [id])
  attendance Attendance[]
  feedbacks  Feedback[]
}

model Subject {
  id          String   @id @default(uuid())
  staffId     String
  staff       Staff    @relation(fields: [staffId], references: [id])
  name        String
  code        String   @unique
  type        String   // THEORY | LAB
  credits     Int
  allocations ClassAllocation[]
}

model Class {
  id          String   @id @default(uuid())
  name        String   // e.g., "2A"
  department  String
  semester    Int
  studentCount Int
  allocations ClassAllocation[]
  students    Student[]
  timetable   Timetable?
  feedbacks   Feedback[]
}

model ClassAllocation {
  id          String   @id @default(uuid())
  classId     String
  class       Class    @relation(fields: [classId], references: [id])
  staffId     String
  staff       Staff    @relation(fields: [staffId], references: [id])
  subjectId   String
  subject     Subject  @relation(fields: [subjectId], references: [id])
  type        String   // THEORY | LAB
  UNIQUE([classId, subjectId, type])
}

model Timetable {
  id          String   @id @default(uuid())
  classId     String   @unique
  class       Class    @relation(fields: [classId], references: [id])
  weekSchedule Json     // { Monday: [{time: "8:30", subject: "Math", room: "101", staff: "John"}] }
  version     Int      @default(1)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

model TimetableSlot {
  id          String   @id @default(uuid())
  timetableId String
  timetable   Timetable @relation(fields: [timetableId], references: [id])
  day         String   // MONDAY, TUESDAY, etc.
  startTime   DateTime
  endTime     DateTime
  subjectId   String
  room        String
  staffId     String
}

model Attendance {
  id        String   @id @default(uuid())
  studentId String
  student   Student  @relation(fields: [studentId], references: [id])
  classId   String
  class     Class    @relation(fields: [classId], references: [id])
  timestamp DateTime @default(now())
  qrCodeId  String?
}

model Feedback {
  id          String   @id @default(uuid())
  studentId   String
  student     Student  @relation(fields: [studentId], references: [id])
  classId     String
  class       Class    @relation(fields: [classId], references: [id])
  staffId     String
  rating      Int      // 1–5
  comment     String   // open-ended
  sentiment   Float?   // 0–1 from NLP
  topics      String[] // ["teaching style", "pacing"]
  createdAt   DateTime @default(now())
}

model Availability {
  id        String   @id @default(uuid())
  staffId   String
  staff     Staff    @relation(fields: [staffId], references: [id])
  date      DateTime
  isAvailable Boolean @default(true)
  reason    String?  // sick, leave, emergency
}

model Notification {
  id        String   @id @default(uuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  message   String
  type      String   // TIMETABLE_SWAP | ABSENCE_ALERT | FEEDBACK_REQUEST
  read      Boolean  @default(false)
  createdAt DateTime @default(now())
}

model Notes {
  id        String   @id @default(uuid())
  classId   String
  class     Class    @relation(fields: [classId], references: [id])
  staffId   String
  staff     Staff    @relation(fields: [staffId], references: [id])
  content   String
  createdAt DateTime @default(now())
}

enum Role {
  STAFF
  HOD
  STUDENT
}
